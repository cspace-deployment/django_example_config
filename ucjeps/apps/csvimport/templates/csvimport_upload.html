Caveat utilizator!

Importing into CollectionSpace is not for the faint of heart. Even though you have a nice, clean "flat file" in the form of a spreadsheet or .csv file you are not out of the woods at all.

It is highly recommended that you read the documentation if you run into even the slightest difficulty!

You are encouraged to experiment first, and to take an incremental approach to getting for file (can CSpace) prepared for each other. There is no harm that can come from using the first two steps below (Count and Validate); however, once you have Counted or Validateed, it will be a challenge to back out -- there is no Undo. The procedure for removing records you have added is described in the documentation above, but it is far from simple.

There is a suggested workflow that may help you sort out the inevitable data bugs efficiently:
  1. Count first (this will ensure the app undertstands your file format);
  2. Validate (which will check the actual content of the columns). These three steps are quick and easy even for large files and can save you trouble going forward.
  3. then try to Upload.
A couple additional pointers:
  1. If you are wondering how to match your columns to CSpace records, try clicking Show Configuration below to see which fields this webapp knows about.
  2. Fill in this form from top to bottom. Pretty much everything needs to be filled in.
  3. Ensure you have all the bits needed before you try: if updating, you'll need to know what columns to specify for matching rows to CSpace records.
